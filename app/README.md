# xOceania web app

A mobile-first, self-contained web app that tells the xOceania story and lets anyone
play with the operator view from their phone. Built for a QR code: scan, and the
pre-dawn oxygen-crash demo opens in the browser.

It is plain static files (no build step, no framework), so it hosts anywhere and
nothing can break live on stage.

## What is in it

| File | Role |
| --- | --- |
| `index.html` | The whole page: hero story, the 30-second film, and the interactive operator demo. |
| `styles.css` | Brand styling (navy palette), mobile-first and responsive. |
| `app.js` | Renders the dissolved-oxygen forecast charts and the interactions (Apply aeration, etc.). |
| `pond.js` | The live overhead pond: sensor pods, aerators with oxygen ripples, fish moving to oxygen, and the drone scanning the grid with per-zone data-custody countdowns. |
| `LOVABLE_PROMPT.md` | Copy-paste prompt for Lovable to build the marketing hero/landing page that links to this demo. |
| `assets/icons.css` | Tabler icon webfont, embedded as base64 so no CDN is needed at runtime. |
| `assets/logo.png` | xOceania logo. |
| `assets/film.mp4`, `assets/poster.png` | The 30-second demo film and its poster. |
| `make_qr.py` | Generates the branded QR code that points at the hosted app. |
| `qr/xoceania_qr.png` / `.svg` | The QR code (currently a placeholder URL, see below). |

## Preview locally

```bash
# from the repo root
python -m http.server -d app 8000
# then open http://localhost:8000
```

Or just open `app/index.html` in a browser.

## Host it (pick one)

The app is the `app/` folder. Any static host works. In rough order of "least friction":

- **Netlify (drag and drop):** go to app.netlify.com/drop and drop the `app/` folder.
  You get a URL in seconds. (Or connect the repo and set base directory to `app`.)
- **Vercel:** import the repo, set the project root / base directory to `app`,
  framework preset "Other". Deploy.
- **Lovable (your plan):** Lovable is React-native and its GitHub sync expects a
  project at the repo root, so the cleanest path for this static app is to let
  Lovable serve it as a static site, or recreate the page in Lovable from this
  design and `index.html`. If you would rather keep editing visually in Lovable,
  tell me and I will hand you a React + Vite + Tailwind version in its own repo.
- **GitHub Pages:** Pages serves from the repo root or `/docs`, not an arbitrary
  `/app` subfolder, so this needs either moving the files or a small Pages Action.
  Netlify or Vercel is simpler for a subfolder.

## The QR code

Once the app is live and you have its URL, regenerate the QR so it points there:

```bash
python app/make_qr.py "https://your-app.lovable.app"
```

That overwrites `app/qr/xoceania_qr.png` (and `.svg`). The QR uses high error
correction with the logo in the centre, so it still scans reliably. For the most
bulletproof version (no centre logo), add `--no-logo`.

The committed QR currently encodes the GitHub repo as a placeholder. Replace it
with the live URL before printing.

## Notes

- Numbers are illustrative, generated by the open-source xOceania simulator
  (`../src`, see the repo README). This is a concept demo, not a live data feed.
- On-screen copy uses no em dashes, matching the deck style.
- To wire it to real telemetry later, replace the hardcoded arrays in `app.js`
  (`MEASURED`, `NOACTION`, `ACTED`) with values from the simulator or live probes.
